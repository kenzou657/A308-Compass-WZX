# AGENTS.md

This file provides guidance to agents when working with code in this repository.

## Ask Mode - Project-Specific Context

### Project Structure Quirks
- **`ti_msp_dl_config.c/h`**: Auto-generated by SysConfig - appears in project but should never be manually edited
- **`empty.syscfg`**: Configuration file that looks simple but controls all peripheral initialization
- **`keil/` directory**: Contains Keil project files (`.uvprojx`), not source code
- **`doc/` directory**: Contest requirements - critical for understanding functional constraints

### Hidden Dependencies
- **SysConfig → Keil Build**: BeforeMake hook runs `syscfg.bat` automatically - if SysConfig changes aren't reflected, rebuild is needed
- **DriverLib → Register Mapping**: DriverLib functions abstract register operations - direct register access will conflict
- **ISR → Main Loop**: ISRs must communicate via flags/queues, not direct function calls to App layer

### Counterintuitive Patterns
- **50Hz Servo Lock**: Not a software constraint - must be configured in SysConfig timer period (20ms)
- **Frame Validation**: 0x55/0xAA headers are protocol-level, not hardware-level - must be checked in UART ISR
- **PID in Main Loop**: Not in ISR - ISR only sets flag, main loop processes control algorithm

### Documentation References
- **Contest Spec**: `doc/2026 年四川省大学生电子设计竞赛 西南石油大学成都校区选拔赛.md`
  - Task 1-4: Basic requirements (line tracking, digit recognition, object transport)
  - Task 5+: Advanced features (autonomous pickup, sequential transport)
  - Size limit: ≤25cm(L)×15cm(W)×15cm(H)
- **DriverLib Docs**: Refer to TI SDK documentation for function signatures
- **Keil Project**: `keil/empty_LP_MSPM0G3507_nortos_keil.uvprojx` defines build settings

### Performance Constraints
- **80MHz MCLK**: Configured in SysConfig, not changeable in code
- **Integer Arithmetic**: Floating point is expensive - use scaled integers (e.g., angle as 0-36000 for 0-360°)
- **ISR Latency**: Keep ISRs minimal to avoid missing interrupts on high-frequency signals

### Debugging Context
- **XDS-110 Debugger**: Built-in on LaunchPad, requires SWD jumpers ON
- **No RTOS**: Bare-metal execution - no task scheduler, use main loop + ISRs
- **Memory Layout**: 32KB SRAM, 128KB Flash - check for stack overflow if behavior is erratic
