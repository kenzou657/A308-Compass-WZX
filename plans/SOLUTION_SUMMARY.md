- ✓ 实现灵活的路径规划系统
- ✓ 支持通过修改角度参数快速适配不同目标区

## 🎯 核心设计要点

### 1. 任务函数接口规范

每个任务需要实现 6 个标准接口：

```c
void TaskX_Init(void);              // 初始化
void TaskX_Run(void);               // 主循环
void TaskX_Stop(void);              // 停止
void TaskX_Reset(void);             // 重置
TaskState_t TaskX_GetState(void);   // 获取状态
bool TaskX_IsSuccess(void);         // 判断成功
```

### 2. 路径规划系统

**核心思想**：将复杂的多阶段运动分解为原子路径点

```c
typedef struct {
    int16_t target_yaw;      // 目标偏航角（°×100）
    uint32_t duration_ms;    // 运动时间（ms）
    uint16_t base_pwm;       // 基础 PWM 占空比
    uint8_t direction;       // 运动方向
    uint32_t pause_ms;       // 停顿时间（ms）
} PathPoint_t;
```

**优势**：
- 灵活组合：支持任意多段路径
- 易于修改：只需改变角度参数
- 可复用：预定义路径表支持快速查询

### 3. 状态机设计

每个任务都是一个独立的状态机：

```
IDLE → INIT → RUNNING → SUCCESS/FAILED/TIMEOUT
```

**特点**：
- 清晰的状态转移
- 支持暂停/恢复
- 完善的超时处理

### 4. 时间控制策略

| 任务 | 时间要求 | 实现方案 |
|------|---------|--------|
| Task 1-2 | ±500ms | 基于 uwTick 计时 |
| Task 3 | ±2s（30±2s） | 动态速度调整 |
| Task 4 | ±3s | 摄像头识别超时 |

## 💡 关键技术点

### 1. 陀螺仪角度控制
- 使用 [`drv_jy61p.h`](src/drivers/drv_jy61p.h) 获取 Yaw 角
- 基于 P 环控制差速，实现方向控制
- 死区设置：|error| < 0.5° 时不动作

### 2. 时间精确控制
- 使用 `uwTick` 系统滴答计数（1ms 精度）
- 每个阶段记录开始时间
- 计算已运行时间，判断是否到达目标时间

### 3. 状态机设计
- 每个任务独立的状态机
- 状态转移条件清晰
- 支持暂停、恢复、停止操作

### 4. 路径规划
- 预定义路径表，支持快速查询
- 支持动态路径修改（通过修改角度参数）
- 支持多段路径组合

## 📞 常见问题

### Q: 如何快速适配不同的目标区？
**A**: 修改路径规划表中的 `target_yaw` 参数即可。例如：
- 目标区1：0°（直行）
- 目标区2：+45°（左转45°）
- 目标区3：+90°（左转90°）
- 目标区4：-45°（右转45°）
- 目标区5：-90°（右转90°）

### Q: 如何处理时间精度问题？
**A**: 使用 `uwTick` 系统滴答计数，精度为 1ms。对于 Task 3（30±2s），可以通过动态调整 PWM 来控制速度。

### Q: 如何调试任务执行？
**A**: 使用 UART 日志输出、OLED 实时显示和 XDS-110 调试工具。详见 [`Implementation_Guide.md`](plans/Implementation_Guide.md) 中的调试技巧。

### Q: 如何处理异常情况？
**A**: 每个任务都有超时检查和异常处理机制。详见 [`System_Architecture_Design.md`](plans/System_Architecture_Design.md) 中的错误处理策略。
