# 按键驱动与任务切换系统 - 集成说明

## 📋 概述

本文档说明如何使用按键驱动和任务切换系统，实现通过K1、K2、K3三个按键来选择和启动不同的任务。

## 🔧 硬件配置

| 按键 | GPIO引脚 | 物理引脚 | 功能说明 |
|------|----------|----------|----------|
| K1 | GPIOB.21 | Pin 20 | 上一个任务 |
| K2 | GPIOB.23 | Pin 22 | 下一个任务 |
| K3 | GPIOB.24 | Pin 23 | 确认/启动/停止 |

**注意**：按键为低电平有效（按下时GPIO读取为0）

## 🚀 使用流程

### 1. 初始化流程

在 [`empty.c`](../empty.c:53) 的 `main()` 函数中：

```c
int main(void)
{
    SYSCFG_DL_init();           // 系统配置（SysConfig生成）
    uart_init();                // 串口初始化
    camera_uart_init();         // 摄像头通信初始化
    OLED_Init();                // OLED显示初始化
    
    Key_Init();                 // 按键驱动初始化
    Key_Logic_Init();           // 按键逻辑初始化
    TaskManager_Init();         // 任务管理器初始化
    
    while (1) {
        Key_Proc();             // 按键处理（10ms周期）
        TaskManager_Update();   // 任务执行
        Camera_Proc();          // 摄像头处理
        OLED_Proc();            // OLED更新（200ms周期）
    }
}
```

### 2. 按键操作说明

#### IDLE状态（任务选择模式）

- **按K1**：切换到上一个任务（8 → 7 → 6 → ... → 1 → 8）
- **按K2**：切换到下一个任务（1 → 2 → 3 → ... → 8 → 1）
- **按K3**：启动当前选中的任务

#### RUNNING状态（任务执行中）

- **按K3**：停止当前任务，返回选择模式
- **K1/K2**：在运行状态下不响应

### 3. OLED显示

#### 选择模式显示

```
┌─────────────────┐
│ Task ID: 1      │  ← 当前选中的任务ID
│ Status: IDLE    │  ← 系统状态
│ K1/K2: Select   │  ← 按键提示
│ K3: Start       │  ← 按键提示
└─────────────────┘
```

#### 运行模式显示

```
┌─────────────────┐
│ Task ID: 1      │  ← 当前执行的任务ID
│ Status: RUN     │  ← 系统状态
│ Running...      │  ← 运行提示
│ K3: Stop        │  ← 按键提示
└─────────────────┘
```

## 🔍 核心模块说明

### 1. 按键驱动层 ([`drv_key.h`](../src/drivers/drv_key.h) / [`drv_key.c`](../src/drivers/drv_key.c))

**职责**：硬件抽象、消抖处理、状态检测

**关键函数**：

- `Key_Init()` - 初始化按键驱动
- `Key_Scan()` - 扫描按键状态（建议10ms周期调用）
- `Key_GetEvent(key_id)` - 获取按键事件（读取后自动清除）
- `Key_GetState(key_id)` - 获取按键当前状态

**消抖机制**：
- 消抖延迟：20ms
- 扫描周期：10ms
- 消抖计数：2次确认

**状态机**：
```
RELEASED → PRESS_DEBOUNCE → PRESSED → RELEASE_DEBOUNCE → RELEASED
```

### 2. 按键逻辑层 ([`key_logic.h`](../src/app/key_logic.h) / [`key_logic.c`](../src/app/key_logic.c))

**职责**：按键事件处理、任务选择状态机

**关键函数**：

- `Key_Logic_Init()` - 初始化逻辑层
- `Key_Logic_Process()` - 处理按键事件（主循环中调用）
- `Key_Logic_GetSystemState()` - 获取系统状态

**事件处理逻辑**：

```c
// IDLE状态
if (k1_event == KEY_EVENT_PRESSED) {
    TaskManager_PrevTask();  // 上一个任务
}
if (k2_event == KEY_EVENT_PRESSED) {
    TaskManager_NextTask();  // 下一个任务
}
if (k3_event == KEY_EVENT_PRESSED) {
    TaskManager_StartTask(); // 启动任务
    g_system_state = SYSTEM_STATE_RUNNING;
}

// RUNNING状态
if (k3_event == KEY_EVENT_PRESSED) {
    TaskManager_StopTask();  // 停止任务
    g_system_state = SYSTEM_STATE_IDLE;
}
```

### 3. 任务管理器 ([`app_task_manager.h`](../src/app/app_task_manager.h) / [`app_task_manager.c`](../src/app/app_task_manager.c))

**职责**：任务注册、切换、执行

**关键函数**：

- `TaskManager_Init()` - 初始化任务管理器
- `TaskManager_StartTask()` - 启动当前任务
- `TaskManager_StopTask()` - 停止当前任务
- `TaskManager_PrevTask()` - 切换到上一个任务
- `TaskManager_NextTask()` - 切换到下一个任务
- `TaskManager_Update()` - 执行当前任务（主循环中调用）
- `TaskManager_GetCurrentTaskID()` - 获取当前任务ID

**任务启动流程**：

```c
void TaskManager_StartTask(void)
{
    const Task_t *task = &g_tasks[g_current_task_id - 1];
    
    task->reset();  // 重置任务状态
    task->init();   // 初始化任务资源
    
    // 记录开始时间
    g_task_start_time = uwTick;
}
```

**任务执行流程**：

```c
void TaskManager_Update(void)
{
    const Task_t *task = &g_tasks[g_current_task_id - 1];
    
    task->run();    // 执行任务主逻辑
}
```

## 📝 任务实现要求

每个任务模块（如 `app_task_1_line_tracking.c`）需要实现以下接口：

```c
void Task1_Init(void);          // 初始化任务
void Task1_Run(void);           // 任务主循环
void Task1_Stop(void);          // 停止任务
void Task1_Reset(void);         // 重置任务状态
TaskState_t Task1_GetState(void); // 获取任务状态
bool Task1_IsSuccess(void);     // 判断任务是否成功
```

**任务执行示例**：

```c
void Task1_Run(void)
{
    // 任务内部状态机
    switch (task_state) {
        case TASK_STATE_INIT:
            // 初始化逻辑
            task_state = TASK_STATE_RUNNING;
            break;
            
        case TASK_STATE_RUNNING:
            // 主要逻辑
            // 例如：循迹控制、数字识别等
            
            if (任务完成条件) {
                task_state = TASK_STATE_SUCCESS;
            }
            break;
            
        case TASK_STATE_SUCCESS:
            // 任务完成后的处理
            break;
    }
}
```

## ⚙️ 配置说明

### SysConfig配置

按键GPIO已在 `empty.syscfg` 中配置：

- GPIO Group: `GPIO_KEYS`
- K1: GPIOB.21 (输入，内部上拉)
- K2: GPIOB.23 (输入，内部上拉)
- K3: GPIOB.24 (输入，内部上拉)

### 时序配置

| 模块 | 调用周期 | 说明 |
|------|----------|------|
| Key_Scan() | 10ms | 按键扫描周期 |
| Key_Logic_Process() | 10ms | 按键逻辑处理 |
| TaskManager_Update() | 实时 | 任务执行 |
| OLED_Proc() | 200ms | 显示更新 |

## 🐛 调试建议

### 1. 按键不响应

- 检查SysConfig中GPIO配置是否正确
- 检查GPIO上拉电阻是否启用
- 使用调试器查看 `Key_GetState()` 返回值
- 检查 `Key_Scan()` 是否被定期调用

### 2. 按键抖动

- 确认 `Key_Scan()` 调用周期为10ms
- 检查消抖计数器是否正确工作
- 可以增加 `KEY_DEBOUNCE_COUNT` 值

### 3. 任务不启动

- 检查 `TaskManager_StartTask()` 是否被调用
- 检查任务的 `init()` 函数是否正确实现
- 使用调试器查看 `g_system_state` 状态

### 4. OLED显示异常

- 检查 `OLED_Proc()` 是否被定期调用
- 检查 `TaskManager_GetCurrentTaskID()` 返回值
- 检查 `Key_Logic_GetSystemState()` 返回值

```
按键按下
    ↓

    ↓
检测GPIO电平变化
    ↓
消抖处理（20ms延迟）
    ↓
生成按键事件（KEY_EVENT_PRESSED）
    ↓
Key_Logic_Process()
    ↓

    ├─ IDLE: K1/K2切换任务，K3启动任务

调用TaskManager接口
    ├─ TaskManager_PrevTask()
    ├─ TaskManager_NextTask()
    ├─ TaskManager_StartTask()
    └─ TaskManager_StopTask()
    ↓
TaskManager_Update() 执行任务
    ↓
Task->run() 任务主逻辑
    ↓
OLED_Proc() 更新显示
```
